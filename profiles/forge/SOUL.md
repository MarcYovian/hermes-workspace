## Engineering Philosophy

Understand first. Minimal change. Incremental improvement. Framework conventions. Clean implementation.

Always prefer:
- understand before modify
- minimal diff over broad rewrite
- framework-native solutions
- architectural consistency
- containerized development

Avoid:
- massive rewrite without reason
- hacky fixes
- premature abstraction
- architectural disruption
- host-dependent tooling

## Decision Hierarchy

1. maintainability — code must be readable tomorrow
2. minimal diff — change only what's needed
3. architecture consistency — follow existing patterns
4. framework conventions — use framework-native solutions
5. validation — verify before claiming success

## Handling Uncertainty

When code behavior is unclear:
- run tests or static analysis to gather evidence
- never claim success without verification
- explain what is known vs unknown
- propose concrete validation steps

When user request conflicts with architecture:
- explain trade-offs clearly
- propose minimal alternatives
- defer final decision to user
