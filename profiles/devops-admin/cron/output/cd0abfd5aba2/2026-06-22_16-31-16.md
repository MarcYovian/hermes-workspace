# Server Health Report (Indonesian Format)

Generate a clean server health report in Indonesian for delivery via messaging platforms. Output ONLY the formatted report structure — no agent commentary, no thinking-out-loud text, no workflow notes.

## When to Use

- Daily scheduled health reports (via cron)
- On-demand health status requests
- Post-maintenance verification

## Report Structure

The final output MUST be exactly this structure with NO additional text:

```
**📊 Server Health Report**
**Tanggal:** [current date and time in Indonesian format]

**⏱ Uptime:** [system uptime from uptime command]
**💻 CPU:** [CPU load percentage and load average]
**💾 RAM:** [used/total in GB with percentage]
**💿 Disk:** [used/total for root filesystem with percentage]
**🐳 Docker:** [running/total containers or 'Not installed']
**❌ Failed Services:** [list of failed services or 'None']
**📦 Pending Updates:** [count of available updates or 'N/A']
**✅ Kondisi:** [Sehat/Perhatian/Kritis]
**📝 Catatan:** [brief observations or 'Tidak ada']
```

## Data Collection

1. **Date/Time**: Use Indonesian locale format (e.g., "Senin, 22 Juni 2026 19:30 WIB")
2. **Uptime**: `uptime -p` for human-readable format
3. **CPU**: 
   - Get load average: `cat /proc/loadavg | awk '{print $1}'`
   - Get core count: `nproc`
   - Calculate percentage: `(load / cores) * 100`
   - Format: "X% (load avg: Y.YY)"
4. **RAM**: 
   - `free -h` to get used and total
   - Format: "X.XG / Y.YG (Z%)"
5. **Disk**: 
   - `df -h / | tail -1 | awk '{print $3 "/" $2 " (" $5 ")"}'`
6. **Docker**: 
   - Check if installed: `command -v docker`
   - If yes: `docker ps -q | wc -l` for running, `docker ps -aq | wc -l` for total
   - Format: "X / Y containers" or "Not installed"
7. **Failed Services**: 
   - `systemctl --failed --no-pager --no-legend | awk '{print $1}'`
   - If none: "None"
8. **Pending Updates**: 
   - `apt list --upgradable 2>/dev/null | grep -c upgradable` or similar
   - If apt not available: "N/A"

## Kondisi (Overall Status)

- **Sehat** (Healthy): All metrics normal
  - Disk < 80%
  - CPU load < 80%
  - No failed services
  
- **Perhatian** (Warning): Minor issues detected
  - Disk 80-90%
  - CPU load 80-100%
  - Pending updates > 20
  
- **Kritis** (Critical): Immediate attention required
  - Disk > 90%
  - CPU load > 100%
  - Failed systemd services
  - Critical services down

## Catatan (Notes)

Include only actionable observations:
- High disk usage with specific partition
- Elevated CPU load with cause if known
- Failed service names
- Unusual container states

If no issues: "Tidak ada"

## Pitfalls

- **NEVER include agent commentary** like "Baik, saya sudah mengumpulkan..." or "Let me check..."
- **NEVER include calculation explanations** like "Kalkulasi:" or debugging text
- **NEVER include workflow instructions** like "To stop or manage this job..."
- **DO NOT add markdown code blocks** around the report — it should be plain formatted text
- **DO NOT explain what you're doing** — just output the report
- CPU percentage calculation must account for core count (load 2.0 on 4 cores = 50%, not 200%)
- Docker commands will fail if Docker isn't installed — check first
- `apt list --upgradable` includes a header line — subtract 1 from count
